schema:accessibilitySummary

A schema.org Thing > Property : accessibilitySummary

Canonical URL: https://schema.org/accessibilitySummary

Check for open issues.

A human-readable summary of specific accessibility features or deficiencies, consistent with the other accessibility metadata but expressing subtleties such as "short descriptions are present but long descriptions will be needed for non-visual users" or "short descriptions are present and no long descriptions are needed".

The accessibility summary was intended (in EPUB Accessibility 1.0) to describe in human-readable prose the accessibility features present in the publication as well as any shortcomings. From EPUB Accessibility version 1.1 the accessibility summary became a human-readable summary of the accessibility that complements, but does not duplicate, the other discoverability metadata.

Up to date definition and exemples are available from

EPUB Accessibility 1.1.

For testing purposes this EPUB has two Accessibility Summary.

  • One written in English with no language tag.
  • One written in French with a lang=fr tag to test if correct language tag is applied to it.

a11y:exemption

An EPUB Vocabulary

Canonical URL: https://www.w3.org/TR/epub-a11y-exemption/

Check for open issues.

The exemption property allows EPUB creators to indicate that an EPUB publication that does not meet accessibility conformance requirements has an exemption under the applicable jurisdiction's laws. Values should be drawn from the approved vocabulary.

Values expected to be one of these types: Text

  • eaa-microenterprise
  • eaa-fundamental-alteration
  • eaa-disproportionate-burden
